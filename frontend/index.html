    Frontend¶

    This page documents the frontend implementation used by PortalRH.

    Overview¶

    The frontend is implemented in React + TypeScript and provides the user interface for authentication, employee lifecycle operations, and reporting flows.

    Technology Stack¶

    • React 19
    • TypeScript
    • React Router
    • Tailwind CSS
    • Axios
    • React Hook Form + Yup
    • Chart.js + React Chart.js 2

    Frontend Location¶

    Project path:

    • frontend/

    Reference page inside frontend module:

    • frontend/page.md

    Detailed Architecture Reference¶

    For a complete breakdown of the frontend module, including:

    • full src/ structure
    • route map from App.tsx
    • detailed responsibilities of each file in src/pages
    • notes about debug/support pages

    use:

    • frontend/page.md

    Local Development¶

    cd frontend
npm install
npm run dev

    Default local URL: http://localhost:3000

    Available Scripts¶

    npm run dev
npm run build
npm run test
npm run lint
npm run lint:fix
npm run type-check

    Environment Variables¶

    Use a .env file in the frontend directory.

    Example:

    REACT_APP_API_URL=http://localhost:8000/api
NODE_ENV=development
REACT_APP_APP_NAME=PortalRH
REACT_APP_VERSION=1.0.0

    Integration Notes¶

    • Ensure backend is running before testing authenticated frontend flows.
    • Keep service modules centralized for API calls and error handling.
    • Keep labels and translations synchronized with i18n files when adding screens.
